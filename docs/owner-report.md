# 오너 리포트 — 디자인 시스템 정합화 사이클

Storybook(React) ↔ Figma가 **같은 토큰·같은 이름·같은 변형**으로 맞물리도록 만든 작업의 결과·사고·부채 정리.

---

## 1. 무엇이 끝났나

### 레퍼런스 시안 반영 (4개 페이지 + ProductCard)
PortfolioPage · HistoryPage · ShopPage · ContactPage를 시안대로 재구성. 스크린샷으로 대조 확인.
반복 요소는 페이지에 박지 않고 **공용 축으로 흡수**했다 — 페이지는 조립만 한다.

### 중복 제거 (컴포넌트 소스 ~7,000줄 규모)
| 없앤 것 | 어디로 |
|---|---|
| FileUpload · ImageUpload | `Upload`의 `preview` 축 |
| CustomerDetailPage | `CustomerDetail`의 `header` 축 + 필드별 `show` |
| AdminBoard | `AdminTable`의 `PRODUCT_COLUMNS` 프리셋 |
| 목록 화면 13개의 뼈대 | `AdminListPage` 셸 (화면은 컬럼·상태·문구만 남는 얇은 프리셋) |
| 폼 화면 3개의 뼈대 | `AdminFormPage` 셸 |
| `admin2.ts` 1,076줄 | 삭제 (아무도 호출하지 않는 죽은 생성기) |

### 모든 텍스트를 밖에서 갈아끼울 수 있게 개방
59개 컴포넌트. 통로는 **`labels` prop 하나**로 통일(`src/shared/labels.ts`가 공용 타입의 단일 출처).
- 우선순위 `개별 prop > labels.* > 기본값` — **기존 API를 하나도 깨지 않았다**(`@deprecated`만 부착)
- 숫자·통화·날짜는 문구가 아니라 포맷 → `formatters`로 분리
- 부분 오버라이드가 나머지 기본값을 지우지 않도록 **그룹 단위 얕은 병합**(`mergeLabels`)

### 코드 ↔ Figma 네이밍 파리티 (오너가 지목한 핵심)
prop 이름 = Figma 속성 이름, CSS 클래스 = 레이어 이름. **`verify-naming` 게이트로 강제.**
- 위반 **1,271건 → 0건** (baseline 689건 전부 소진)
- `buildSet` 헬퍼가 4개 파일에 복사돼 규약 위반을 **4벌씩 증식**시키고 있었다 → 공용 모듈 한 벌로 통합
- 유령 boolean 231개, 유령 INSTANCE_SWAP 24개 제거(React에 대응이 없는데 Figma에만 있던 것)

### 어드민 사이드바 재편 + 회사관리 화면 신규
오너 지정 6그룹 구조. 없던 화면 2개(회사소개 관리·연혁 관리)를 셸 프리셋으로 신설(**새 CSS 0줄**).

## 2. 🔴 사고 2건 (재발 방지 조치 완료)

### 사고 1 — 공유 워크트리에서 `git stash`
한 에이전트가 "게이트 기준선을 잡겠다"며 `git stash`를 실행 → **다른 에이전트 53개 파일의 작업이 HEAD로 되돌아갔다.**
**되돌아간 파일도 타입체크는 통과**하므로(원본이 유효하니까) 아무도 알아채지 못한 채 "완료" 보고가 나갔다.

- **복구**: stash에서 되돌아간 53개만 골라 비파괴 복원. 커밋 안전지점이 있어 피해 0.
- **재발 방지**: `CLAUDE.md §0-0` — **git 상태를 바꾸는 명령 전면 금지**. 커밋·푸시는 오케스트레이터만.

### 사고 2 — 게이트가 전부 초록인데 Figma 화면이 깨져 있었다
Figma 세트의 속성 이름을 규약대로 개명하자 화면의 `inst()` 오버라이드가 **조용히 끊겼다.**
`inst()`는 없는 속성 이름을 **경고만 하고 무시**하기 때문이다.

- **실제 피해**: `ProductCard.Price` 개명 → **상품 카드 10장이 전부 세트 기본가로 렌더**(세일가 소실).
  `SortBar.Count` → **상품 10장짜리 쇼핑 화면이 "6개"를 표시**(이건 HEAD부터 이미 깨져 있었다).
- **왜 못 잡았나**: **5개 게이트가 전부 초록이었다.** 사람이 눈으로 21건을 고쳤고, **그러고도 6건을 놓쳤다.**
- **조치**: `scripts/verify-screen-props.mjs` 신설(S1~S5). 결함 주입 7종 전부 잡는 것을 확인.
  파싱 못 한 호출은 **경고가 아니라 실패**(조용한 통과 금지).
- **교훈**: **개명은 규약 준수인 동시에 사고다.** `verify-naming`이 초록으로 바뀌는 바로 그 순간 화면이 깨진다.

| 엑셀 버튼 라벨 소실 | `ToolbarActions`가 아이콘 전용이라 라벨이 사라짐 | 라벨 있는 Button으로 되돌림 |

## 5. 발견한 실제 결함 (작업의 부산물)

- **AdminChart의 canvas가 스크린리더에 완전히 침묵**하고 있었다 → `ariaLabel`/`ariaDescription`
- **Pagination 숫자 버튼에 접근성 이름 없음** — `"3"`이 무엇의 3인지 알 수 없었다 → `"3페이지"`
- **ProductDetail의 주문상태 톤 규칙이 한국어 리터럴 비교**(`'취소' || '환불'`)로 로직에 박혀 있었다 → 번역하면 규칙이 깨지는 구조. `labels`로 분리
- **Figma가 코드에 없는 것을 그리고 있었다**:
  - `Popover`가 `top` 배치 변형을 그림 — **React Popover는 아래로만 열린다**
  - `Alert` 세트가 `title`+`message` 두 줄 — React는 `label` 한 줄
  - `Radio` 세트가 라디오 **1개**를 그리며 코드에 없는 축 2개를 세우고 정작 `direction`은 빠뜨림
  - KR 필드의 `state: filled` 변형이 사실 `success`를 그리고 있었다
  - Carousel·Calendar·픽커의 `Prev`/`Next` INSTANCE_SWAP이 **아무 레이어에도 바인딩되지 않은 유령**이었다
- **사이트 GNB 메뉴가 두 파일에 중복 선언**돼 서로 어긋나 있었다

## 6. 남은 부채 (오너 판단 필요)

| 부채 | 내용 | 해소 방법 |
|---|---|---|
| **변형 폭발 (선재 문제)** | `DS/Button` **432변형**(variant6 × appearance3 × size3 × disabled2 × fullWidth2 × iconOnly2). ImageCard 72, TextField 48 | 축 일부를 BOOLEAN/TEXT로 내리거나 세트 분리 |
| **`variantItem` 도구 결함** | 문서 생성 헬퍼가 VARIANT 축만 이름으로 설정 가능. BOOLEAN·TEXT는 전체 키 필요 → 규약대로 고치면 **문서에서 '푸터 있는 카드' 그림이 사라진다** | `propKeys`로 이름을 풀게 고치면 자동 해소 |
| **allowlist 중 "연기"된 항목** | 대부분(400여 건)은 정당한 사유(Figma에 숫자·배열·ReactNode 타입 없음 / 데이터라서 prop이 아님). 다만 31건은 **"표현 불가"가 아니라 "모양 불변 제약 때문에 연기"** | `until` 필드를 달고 세트 확장 배치 예약 |
| **`components.ts`가 검증 대상 밖** | 도달 불가(`ui.html`이 `components:false` 하드코딩)라 당장 위험은 없지만 **검사받지 않는 생성기가 트리에 남아 있다** | 되살리려면 네이밍 검사에 먼저 편입 |
| **`EmptyState.kind` 8종 미구현** | Figma에 반영하려면 placeholder 8종을 벡터로 이식해야 하고, 그러면 그려지는 아이콘이 바뀐다(모양 불변 위배) | 별도 배치 |
| **AvatarGroup 드리프트** | Figma md=36px, CSS는 32px | 파리티 배치 |
| **12개 목록 화면의 `labels.table` 버킷** | 셸은 열렸지만 화면이 자기 타입에 `table` 버킷을 안 만들어 **바깥 호출자가 표 크롬 문구까지 닿지 못한다**(화면당 2줄이면 끝) | 후속 배치 |
| **취소된 병합 2건** | 소형 병합(SortBar·Timeline·Dialog·Tag)과 콘텐츠 목록 3개 마이그레이션이 취소된 채로 남음 | 오너 결정 |

## 7. 게이트 (현재 전부 통과)

```bash
npx tsc -p tsconfig.app.json --noEmit      # 앱 타입체크
cd figma-plugin && npx tsc --noEmit         # 플러그인 타입체크
node figma-plugin/scripts/build.mjs         # 번들 빌드
node scripts/verify-parity.mjs              # 토큰·아이콘·로고·커버리지
node scripts/verify-mapping.mjs             # props ↔ Figma 매니페스트
node scripts/verify-naming.mjs              # 코드 이름 ↔ Figma 속성/레이어 이름
node scripts/verify-screen-props.mjs        # 화면 inst() ↔ 세트 속성 (신설)
```

**게이트를 만들 때는 결함을 일부러 주입해 실제로 잡는지 확인했다.** 그렇게 해서 사람이 놓친 6건을 더 찾았다.
"고쳐서 통과"와 "안 봐서 통과"는 다르다.
